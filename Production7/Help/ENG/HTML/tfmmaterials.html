The main grid is simple and self-defining. there is Name and model of the product displayed along with measurement unit, barcode and user selection of pre-defined prices.

By right-clicking on the list you'll be presented with the following menu:

Add - shows add/edit product form with default values fields to start defining a new product

Add using current - A quick way to add a new product similar to selected one. All the bulk of data will be copied from the definition of the selected product as the base. You have to give it a new name or model and click OK in simple case.

Delete - if "Recycling bin" feature wasn't turned off (it's on by default) then selected list of products will be moved into it. Otherwise they will be removed permanently. Note the fact that if some product was included into any of the documents then it cannot be removed. This way. if you want to get rid of it's name in the list you should place it into archive mode.

Refresh - reloads list of products. This is needed for multi-seat environment in case another person was made any modifications to the list.

Product movement details - Displays the "Product movement details" form that present detailed picture of when and where this product items was transferred.

Product reserve details - Displays the "Product reserve details" form that lists how much and by what means items was reserved

Find in warehouse - Jumps to warehouse view for this product

Move to archive - Marks product(s) as archived. You need this in case when those product(s) are obsolete but was included in some documents that should be preserved.

Display archived products - Toggles displaying of archived products.

Web Stores - Submenu containing quick means to update the state of linked web-stores. This item is available with "Web-Stores" license option and displayed only if you had defined a web-store in the internet section.

Set up columns - Let you specify what columns you want to see in this list. Brings up the "Set up columns" form.

Edit - let's edit this product or several selected products at once. In later case only the common features as pricing, etc will be available for editing.

On the first you also see the basic details of the selected product, but with some additional pieces:

Product group tells where this particular product belongs to and default warehouse is the starting place for the incoming lots and outgoing sales. It is just default place to store or look, not a mandatory setting.

The details is obvious too at the top of a few things:

Weight and volume used just for the storage and transporting reference.

Minimum stock level set the lower limit below which there will be warning issued to make new order to the supplier.

The "Pricing" section displays summary about the sales pricing schemes. It is a list of user-defined pricing categories with their respective methods to define the output values. To better understand how the pricing categories work, please refer to marked help section.

As you can see, it is very easy to define a new product group or category just by right-clicking at the tree node where you want to add this new sub-category.

You'll then be presented with the "Add/Edit Product group" form similar to the "Add/Edit Product" form to complete it's description and details

And the last summary tab on the lower pane is "Substitute products":

This useful feature of "substitute products" can help a point of sales people to quickly present different choices of similar to the already chosen or desired product(s) to the client in case of doubts, curiosity or just plain out-of-stock condition.
