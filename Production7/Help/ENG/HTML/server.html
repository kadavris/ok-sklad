The OK-Server application should be automatically launched by request from the clients application.

To accomplish that task there exists another viable component - the Borland Socket Server.

It is third party application that should be always running on the server computer to wait and process the requests from connecting client applications.

You can see the OK-Server and Socket Server down in the "Tray" area of your program panel as show on this picture.

When you right-click on the server's tray icon it will bring the pop-up menu.

•About displays standard type of "About this software" window where you can check the properties of installed Server application along with your registration information.
•Options will show you a form described in details later.
•Server log is useful for your technical staff to quickly check for problems and error if any occur.
•Change server's key - the green check mark indicates that registration key is valid. Otherwise it will display red mark along with "Set server's key" message.
Using this menu option you can actually change the registration key then.
•Exit - does force shutdown of the server. Use with caution as it can damage the data being written into the database by any of still connected clients.

The options for the server application is simple:

•Maximum log size - specify the limit for the logging facility. The log file is used for informative purposes to store the Server's messages that might be of interest when problems may arise. As this can grow to a large extent over a working period it is a good idea to limit it's size by removing the oldest event messages and keeping only the late ones. One megabyte is a default choice and potentially very high limit that barely will be touched ever.
•Language let's you change the interface language. Use it when it is working in your foreign offices where local employees prefer to have the messages displayed in their native language.

Check for dead connections section allow you to deal with clients connecting remotely via Internet or similar unreliable in general medium. If you set to check activity of the clients then one that did not sent any request or update to the database for the given time range will be forcibly disconnected to maintain low recourse usage by the server application.

It will also unblock new connection requests because of limit imposed by your purchased license.

So "check every" will instruct server to go trough the list of connected clients and look up the last time each of them did something. If some of them was inactive more than stated in "Maximum time to keep.." then this client will be disconnected immediately.

The button with "Click to re-enable automatic Socket Server startup" caption will try to fix the case when the Borland Socked Server is not automatically started upon the Windows startup and therefore the clients cannot get access to the OK-Server application that runs only by Borland Socked Server when it detecting incoming connection from client.
